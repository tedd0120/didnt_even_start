# give-up-list 规范

## 目的
明确首页放弃清单的展示内容、信息结构与空状态表达，并规定新增入口的可发现性、交互方式与跳转行为，确保用户能快速添加放弃事项。
## 需求
### 需求：放弃清单展示
系统必须在首页展示所有未达成的放弃事项，并包含时间信息与置顶排序。

#### 场景：查看放弃事项
- **当** 用户进入首页
- **那么** 系统展示每条放弃事项的标题、原本打算时间与记录时间
- **并且** 被置顶的事项在列表顶部显示

### 需求：添加入口
系统必须在首页提供醒目的新增入口以创建新的放弃事项。

#### 场景：进入添加页
- **当** 用户点击新增入口
- **那么** 系统进入添加放弃事项页面

### 需求：放弃事项管理
系统必须允许用户通过列表右滑手势对放弃事项进行置顶、删除（弹窗确认）与转移到达成清单。

#### 场景：右滑操作
- **当** 用户在放弃清单中对某事项右滑
- **那么** 系统展示置顶、转移与删除操作

#### 场景：删除确认
- **当** 用户选择删除操作
- **那么** 系统弹出确认提示并在确认后删除该事项
