# badge-wall 规范

## 目的
明确勋章墙的展示内容与信息结构，定义勋章解锁阈值、展示方式与提示行为，并规定勋章解锁结果在本地的持久化要求与恢复路径。
## 需求
### 需求：勋章墙展示
系统必须在勋章墙展示个人信息区域，并提供海报显示开关。

#### 场景：展示个人信息
- **当** 用户进入勋章墙页面
- **那么** 系统展示头像与昵称（若已设置）以及海报显示开关

#### 场景：切换海报显示
- **当** 用户切换“海报显示头像与昵称”开关
- **那么** 系统保存该设置并立即更新海报预览

### 需求：勋章解锁规则
系统必须按放弃事项数量解锁普通勋章与隐藏勋章，并记录其等级。

#### 场景：解锁普通勋章（条件公开）
- **当** 用户累计放弃事项达到以下条件
- **那么** 系统解锁对应普通勋章并标记其等级
  - 铜：入门躺平奖（1 条）
  - 铜：拖延起步者（3 条）
  - 铜：计划搁置者（5 条）
  - 银：咸鱼进阶者（10 条）
  - 银：沙发守护者（20 条）
  - 银：懒人代表作（30 条）
  - 金：躺平大师（50 条）
  - 金：省力宗师（80 条）
  - 金：梦想搁置王（120 条）

#### 场景：解锁隐藏勋章（条件隐藏）
- **当** 用户累计放弃事项达到以下条件
- **那么** 系统解锁对应隐藏勋章并标记其等级
- **并且** 在勋章墙内不显示其名称与达成条件
  - 铂金：隐秘一号（200 条）
  - 铂金：隐秘二号（300 条）
  - 钻石：隐秘三号（500 条）
  - 钻石：隐秘四号（800 条）
  - 暗物质：隐秘五号（1200 条）
  - 暗物质：隐秘六号（2000 条）

### 需求：勋章本地持久化
系统必须将已解锁勋章持久化到本地存储，以便重启后恢复。

#### 场景：重启后查看勋章
- **当** 用户重启应用并进入勋章墙
- **那么** 系统仍能展示已解锁勋章
