# add-give-up 规范

## 目的
明确新增放弃事项的输入字段、校验与默认时间行为，并定义提交后的治愈反馈与勋章提示的触发时机，同时规定相关数据的本地持久化要求。
## 需求
### 需求：新增放弃事项
系统必须允许用户创建一条新的放弃事项，包含标题、可选的放弃理由，以及原本打算做的时间，并记录创建时间。

#### 场景：提交有效的放弃事项
- **当** 用户填写标题并提交
- **那么** 系统保存该事项并记录创建时间

#### 场景：未填写原本时间
- **当** 用户未填写原本打算做的时间并提交
- **那么** 系统使用当前时间作为该字段的默认值

### 需求：提交反馈
系统必须在提交后展示治愈/幽默的提示，并弹出新解锁的勋章（如有）。

#### 场景：提交后的动画反馈
- **当** 用户成功提交放弃事项
- **那么** 屏幕出现治愈/幽默提示的动画反馈

#### 场景：解锁勋章
- **当** 用户提交放弃事项触发勋章解锁
- **那么** 系统弹出对应的勋章提示

### 需求：本地持久化
系统必须将新增的放弃事项持久化到本地存储，以便重启后恢复。

#### 场景：重启后查看
- **当** 用户重启应用并进入首页
- **那么** 系统仍能展示之前保存的放弃事项
