# Role
你是一名精通 React Native (Expo) 的资深移动端架构师和 UI 设计师。你的任务是辅助我从 0 到 1 开发一款 iOS App。

# Project Overview
应用名称：“甚至没有开始” (Didn't Even Start)
核心概念：这是一个反向的 Todo List。传统的 Todo List 记录你要做的事，而这个 App 记录用户“打算做但最后放弃了”的事情。
核心价值：反内卷，治愈精神内耗，通过幽默的方式缓解用户的焦虑。

# Tech Stack Constraints (Strict)
1. Framework: React Native + Expo (使用最新 SDK 50+)。
2. Language: TypeScript (必须使用强类型)。
3. Navigation: Expo Router (使用 app/ 目录结构的文件路由)。
4. Styling: React Native StyleSheet 或 NativeWind (Tailwind CSS)，风格需保持一致。
5. Storage: AsyncStorage (用于本地持久化存储放弃清单和勋章数据)。
6. Icons: 使用 @expo/vector-icons (Feather 或 Ionicons)。

# Functional Requirements (MVP)
1. **首页 (The Give-up List)**:
   - 展示一个列表，列出所有“已放弃”的事项。
   - 每一个卡片应包含：放弃的事情（标题）、原本打算做的时间、放弃理由（选填）。
   - 添加按钮（大大的悬浮按钮）：点击添加新的放弃事项。

2. **添加页面 (Let it Go)**:
   - 输入框：本来想做什么？（例如：去健身房）
   - 输入框：为什么放弃？（例如：床太舒服了）
   - 提交后的反馈动画：屏幕飘过“太棒了，又省力了”等治愈/幽默的提示，并弹出一个勋章。

3. **勋章墙 (Hall of Shame/Fame)**:
   - 展示用户获得的“躺平勋章”。
   - 勋章逻辑：每放弃 1 件事获得“入门躺平奖”；放弃 10 件获得“咸鱼之王”等。

# UI/UX Design Guidelines
- **Vibe**: 放松、治愈、极简、幽默。
- **Color Palette**: 
  - 背景：暖米色 (#F5F5DC) 或 柔和的深色模式。
  - 主色：低饱和度的绿色（象征治愈）或慵懒的橙色。
  - 字体：圆润、无衬线字体，字号偏大，阅读轻松。
- **Interaction**: 操作要极其简单，不要给用户任何压力。
